> An end-to-end Machine Learning web application that recommends books based on  
> content similarity, collaborative filtering, and a hybrid of both — powered by  
> **FastAPI**, **React**, and **scikit-learn**.

## 🚀 How to Run

### Step 1 — Clone & Install Python Dependencies

```bash
git clone https://github.com/your-username/book-recommendation-system.git
cd book-recommendation-system

# Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate        # macOS/Linux
venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt
```

### Step 2 — Run the Jupyter Notebook

This step **must be done first** — it trains the model and saves the pickle files.

```bash
cd notebook
jupyter notebook book_recommendation_model.ipynb
```

Run all cells (Kernel → Restart & Run All).  
Three `.pkl` files will be saved to `backend/`:
- `hybrid_similarity.pkl`
- `books_data.pkl`
- `pivot_table.pkl`

### Step 3 — Start the FastAPI Backend

```bash
cd backend
uvicorn main:app --reload --port 8000
```

- API base URL: `http://localhost:8000`
- Swagger docs: `http://localhost:8000/docs`
- ReDoc:         `http://localhost:8000/redoc`

### Step 4 — Start the React Frontend

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:3000` in your browser.

## 🔌 API Reference

### `GET /recommend`

Returns top-N book recommendations.

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `book_name` | string | ✅ | — | Full or partial book title |
| `n` | int | ❌ | 5 | Number of results (1–20) |
| `method` | string | ❌ | `hybrid` | `hybrid` \| `content` \| `collaborative` |

**Example Request**
```
GET http://localhost:8000/recommend?book_name=1984&n=5&method=hybrid
```

**Example Response**
```json
{
  "input_book": "1984",
  "method": "hybrid",
  "recommendations": [
    {
      "title": "Brave New World",
      "author": "Aldous Huxley",
      "genre": "Dystopia",
      "similarity_score": 0.8923
    },
    {
      "title": "Fahrenheit 451",
      "author": "Ray Bradbury",
      "genre": "Dystopia",
      "similarity_score": 0.8741
    }
  ]
}
```

### `GET /books`

Returns all available book titles.

```
GET http://localhost:8000/books
```

### `GET /health`

Returns server and model status.

```
GET http://localhost:8000/health
```

## 🤖 How the ML Works

```
User Input: "1984"
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│              Content-Based Filtering                        │
│  TF-IDF(author + genre) → cosine_similarity(book_A, book_B) │
└─────────────────────────────┬───────────────────────────────┘
                              │  0.5×
┌─────────────────────────────▼───────────────────────────────┐
│              Collaborative Filtering                        │
│  pivot_table(books × users) → cosine_similarity(row_A,row_B) │
└─────────────────────────────┬───────────────────────────────┘
                              │  0.5×
                              ▼
                   Hybrid Similarity Score
                              │
                              ▼
              Top-5 Most Similar Books → JSON Response
```

## ⚠️ Limitations

- Dataset is a sample (30 books). Replace with the [Book-Crossing dataset](https://www.kaggle.com/datasets/ruchi798/bookcrossing-dataset) for production.
- Cold-start problem: new books/users not in the training set can't be recommended.
- Collaborative filtering is limited by sparse rating data.
